# 輔大畢業學分查核與選課推薦系統 - AI 開發脈絡指南 (AI Context)

這份文件是專門為了讓 AI（如 Cursor、GitHub Copilot、ChatGPT 等）能快速理解本專案架構與商業邏輯所準備的上下文指南。若要進行後續開發，請先閱讀本文件。

## 1. 專案概述 (Project Overview)
本專案為輔仁大學學生設計，旨在解決學生手動計算畢業學分與查詢課程的痛點。系統會自動爬取學生的「歷年成績」與「畢業檢核表」，與校內的「全校課程清單」進行比對，透過本機 Python GUI 提供精準的學分缺口分析、視覺化圓餅圖，以及智慧化的下學期選課推薦。

## 2. 技術棧 (Tech Stack)
* **後端 / 核心邏輯:** Python 3.12+
* **桌面應用程式介面:** `customtkinter` (深色模式、現代化 UI)
* **資料庫:** MySQL (資料庫名稱: `graduation_db`)
* **網路爬蟲:** 
  * `selenium` (一般爬蟲，如 SIS 成績)
  * `undetected-chromedriver` (用於繞過 Cloudflare 驗證的畢業檢核表爬取)
  * `BeautifulSoup4` (HTML 內容解析)
* **數據分析與報表:** `matplotlib` (圓餅圖), `fpdf2` (PDF 匯出)

## 3. 核心檔案與系統架構 (Core Architecture)
系統分為「爬蟲資料蒐集」與「GUI 運算呈現」兩大部分：

* **`test1.py`** (歷年成績爬蟲)
  * 登入輔大 SIS 系統，抓取學生歷年修課紀錄。
  * 資料寫入至 MySQL 的 `FJU_Personal_Grades` 資料表。
* **`test2.py`** (畢業檢核表爬蟲)
  * 登入學習輔導與預警系統，抓取校定/院系必修的「已修/尚未修課」狀態。
  * 資料寫入至 `FJU_Graduation_Check` 資料表，作為推薦系統最高權重（必修缺口）依據。
* **`get_school_info/scrape_general_edu.py`** (通識課程爬蟲)
  * 爬取大綱系統，自動翻頁獲取通識課的「開課單位」並記錄於 `FJU_GenEd_Departments`。
* **`test_project/gui.py`** (主系統介面)
  * 核心展示層，讀取所有 DB 資料並進行邏輯運算。
  * 包含條件過濾、學分數加總、Matplotlib 圓餅圖渲染、推薦課程演算法與 PDF 報表匯出。

## 4. 資料庫架構 (Database Schema)
* **`FJU_Courses`**: 存放輔大當學期全校所有的開課資訊。
* **`FJU_Personal_Grades`**: 存放使用者的歷年成績與取得學分。
* **`FJU_Graduation_Check`**: 存放使用者的畢業檢核條件與缺口。
* **`FJU_GenEd_Departments`**: 存放通識課程的進階分類與開課單位。

## 5. 關鍵商業邏輯與防呆機制 (Key Business Logic & Rules)
AI 在修改程式碼時，必須嚴格遵守以下邏輯與防呆機制：

1. **同名課程顯示 (GROUP BY)**
   * 學校可能有 10 堂同名的「統計學」（不同老師）。在 GUI 中，讀取 `FJU_Courses` 時會使用 `GROUP BY course_name, credits` 進行合併，避免畫面過於冗長。
2. **自動打勾與學分比對 (Auto-Check & Credit Matching)**
   * 系統會比對 `FJU_Personal_Grades` (及格課程) 與 `FJU_Courses`。
   * **比對條件**：不僅 `course_name` 要相同，`credits` (學分數) 也必須完全吻合，才會自動將該課程打勾。
3. **學分數防重複計算 (Prevent Double Counting)**
   * 若使用者在 GUI 勾選了多堂同名課程（或系統因為某種原因勾選了多堂），`gui.py` 內的 `calculate_credits` 函式使用了 `counted_course_names` 集合來防呆，確保「同名課程的學分數只會被累加一次」。
4. **重複修課採計最高分**
   * 若學生重複修習同一門課，在從資料庫讀取成績時，系統會自動比對並保留「最高分」的紀錄，以此判斷是否過關與顯示成績。
5. **已修課程置頂**
   * 在 GUI 過濾器 (`apply_filter`) 中，系統會將被判定為「已修過關」的課程排序權重拉高，確保它們永遠顯示在列表最上方。

## 7. 專案願景與目標 (Product Goal & Sprint Goals)

**Product Goal (最終產品願景)：** 
建立一個全自動化的輔大畢業學分查核與智慧選課推薦系統。透過自動化爬蟲技術擷取學生歷年成績與畢業檢核表，精準分析學分缺口（如必修、通識與畢業門檻），並結合現代化 GUI 介面提供動態且客製化的選課推薦與視覺化進度追蹤，徹底解決學生手動精算學分與規劃課程的痛點。

**Sprint 1 Goal (本期衝刺目標)：** 
完成系統核心基礎架構與資料自動化擷取。主要實作：建立穩定的 MySQL 資料庫（規劃課程與成績資料表 Schema）、開發 `test1.py` 自動登入 SIS 抓取個人歷年成績，以及開發 `test2.py` 突破防護機制自動爬取畢業檢核表的「已修/未修」項目，完成初步的資料收集與整合。

## 8. 敏捷開發與使用者故事 (Agile & User Stories)

### Epic (史詩)
**建立一個全自動化的輔大畢業學分查核與選課推薦桌面應用程式**
為幫助輔大學生省去手動比對學分與查找課程的時間，系統需自動抓取歷年成績與畢業門檻，提供視覺化的學分分析與智慧化的選課推薦，讓學生能輕鬆規劃畢業藍圖。

### User Story (使用者故事)
1. **作為一個輔大學生**，我希望系統能自動登入 SIS 抓取我的歷年成績並採計最高分，**以便**我不必手動輸入。
2. **作為一個輔大學生**，我希望系統能自動比對我的畢業門檻與通識分類，**以便**我能清楚知道還有哪些學分尚未完成。
3. **作為一個面臨選課的學生**，我希望系統能根據我缺少的學分推薦合適的課程，**以便**我能高效率地選課。
4. **作為一個需要規劃學習進度的學生**，我希望能在系統內看到圖表分析與匯出 PDF 報表，**以便**我可以追蹤自己的畢業進度。
5. **作為一個使用者**，我希望系統擁有現代化且流暢的桌面介面，並能在清單上將已及格課程置頂顯示，**以便**我能有良好的操作體驗。

## 9. 行為驅動開發規格 (BDD Specifications)

### Feature 1: 自動爬取與整合歷年成績
**Feature (功能)：** 學生歷年成績自動爬取與匯入
**As a (身分為)：** 輔仁大學的學生
**I want (我想要)：** 系統能夠自動登入「輔大 SIS 學生資訊系統」並抓取我過去修過的課程與成績
**So that (以便於)：** 我不需要手動一筆一筆輸入已修學分，且系統能確保成績資料的準確性。

* **Scenario: 成功抓取並寫入最高成績**
  * **Given (假設)：** 學生已經在 GUI 登入畫面輸入正確的帳號密碼，且本機 MySQL 資料庫已啟動。
  * **When (當)：** 系統執行 `test1.py` 並成功登入 SIS 進入「成績查詢」頁面。
  * **And (且)：** 學生有重複修過同一門課（例如第一次不及格，第二次及格）。
  * **Then (那麼)：** 系統應該將所有成績爬取下來，並在寫入資料庫判斷時，自動保留該科目的「最高成績」。
  * **And (並且)：** 將該最高成績標記為及格/不及格，並顯示「個人成績抓取完成！共成功寫入 N 筆成績」。

### Feature 2: 畢業門檻缺口比對
**Feature (功能)：** 畢業學分檢核表爬取與比對
**As a (身分為)：** 準備規劃下學期課程的學生
**I want (我想要)：** 系統能自動登入「輔大學習輔導與預警系統」，分析修業科目檢核表
**So that (以便於)：** 我能一目了然地知道自己還欠缺哪些「必修」或「通識」學分才能畢業。

* **Scenario: 辨識尚未修課的必修項目**
  * **Given (假設)：** 學生已經在系統內留存了個人的帳號密碼。
  * **When (當)：** 系統執行 `test2.py`，導航至「畢業學分檢核」頁籤。
  * **And (且)：** 系統在網頁中掃描到標示為「尚未修課」或「不及格」的必修課列。
  * **Then (那麼)：** 系統必須將該筆必修課程紀錄進 `FJU_Graduation_Check` 資料表，狀態標示為未完成。
  * **And (並且)：** 在推薦系統運算時，將這些未完成的必修課程設定為「最高推薦權重」。

### Feature 3: 智慧選課推薦與 GUI 介面顯示
**Feature (功能)：** 智慧選課推薦與動態圖表
**As a (身分為)：** 正在為選課苦惱的學生
**I want (我想要)：** 透過現代化桌面介面查看我缺少的學分，並獲得系統自動組合的課程推薦清單
**So that (以便於)：** 我能以最有效率的方式補齊畢業所需的學分數。

* **Scenario: 動態補齊學分推薦**
  * **Given (假設)：** 學生進入了 Python GUI 主畫面，且系統已載入使用者的歷年成績與所有輔大課程資料。
  * **When (當)：** 學生點擊「分析缺少學分與推薦課程」的功能按鈕。
  * **And (且)：** 系統計算出學生目前還差 6 學分的「選修」以及 1 門「必修」。
  * **Then (那麼)：** 系統應該在介面上顯示已過關課程的打勾狀態與成績（如 `🏆 85`）。
  * **And (並且)：** 系統應自動從未修課程清單中，推薦 1 門該必修課以及總計 6 學分的選修課組合，並將結果呈現在介面上供學生參考。
